# RP2040 Temperature Monitor

RP2040内蔵温度センサを使った学習用ベアメタルファームウェアプロジェクト。
DMA、I2C、リングバッファ、デュアルコアの動作を実機で観察する。

## 目的

本プロジェクトは個人の学習目的で、以下を実機で動作させながら理解することを目指す。

- RP2040のSVDからレジスタ定義を生成し、ゼロから動かすベアメタル開発
- レイヤードアーキテクチャ（App / Service / Driver / HAL / Register / Hardware）
- ADC + DMAによるCPU介入のないサンプリング
- I2C経由のLCD1602表示
- リングバッファによるサンプル蓄積と統計処理
- デュアルコア（Cortex-M0+ ×2）での責務分担
- フェイルセーフ（I2C通信障害の検出と通知）

## 構成

第1フェーズはC言語、第2フェーズでRust移植を行う。
要件・設計ドキュメントは sphinx-needs で管理。

### ディレクトリ

```bash
    rp2040-temp-monitor/
    ├── .devcontainer/   開発環境(docs用)
    ├── docs/            要件・設計ドキュメント
    │   ├── common/      言語非依存(システム要件、アーキテクチャ、SW要件)
    │   ├── c/           C固有の詳細設計(将来追加)
    │   └── rust/        Rust固有の詳細設計(将来追加)
    ├── diagrams/        図(PlantUML)
    ├── firmware-c/      C実装(段階2以降で追加)
    ├── firmware-rust/   Rust実装(段階7以降で追加)
    └── svd/             RP2040 SVDファイル(段階2で追加)
```

## 開発環境

VSCode + Dev Containersが前提。リポジトリを開いて`Reopen in Container`を選択すれば環境が立ち上がる。

## ドキュメントのビルド

```bash
make html
```

成果物は`_build/html/index.html`で閲覧できる。

### クリーンビルド

要件の構造変更（リレーション追加・削除など）を行った場合、キャッシュの影響でリレーションが正しく表示されないことがある。
その場合はクリーンビルドする：

```bash
make rebuild
```

または：

```bash
make clean && make html
```

## ローカルサーバで確認

```bash
make serve
```

`http://localhost:8080`でブラウザから確認できる。

## 要件の規模

## 要件の規模

- システム要件: 6個(SYS_001〜006)
- アーキテクチャ要件: 7個(ARC_001〜007)
- ソフトウェア要件: 40個(SWR_001〜040)
- ADR: 2個(ADR-0001, ADR-0002)
