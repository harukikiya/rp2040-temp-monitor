# プロジェクト概要

> **位置づけ**：本ドキュメントは本プロジェクト全体の概要と、ASPICEプロセス対応の方針を示す。
> 各設計文書を読む前に、まず本ドキュメントを読むことを推奨する。

## プロジェクトの目的

RP2040の内蔵温度センサを使った学習用ファームウェアプロジェクト。
DMA、I2C、リングバッファ、デュアルコア活用の動作を実機で観察することを目的とする。
実装言語はC言語（第1フェーズ）、その後Rust（第2フェーズ）。

## ハードウェア前提

本プロジェクトは以下のハードウェア構成を前提とする：

- **MCU**：Raspberry Pi Pico（RP2040、Cortex-M0+ デュアルコア、264KB SRAM、外付け 2MB QSPIフラッシュ W25Q16JV）
- **温度センサ**: RP2040 内蔵温度センサ(ADC4 経由)
- **表示**：I2C接続のLCD1602（16文字×2文字 LCD）
- **状態表示**：オンボード LED（GPIO 25）
- **書き込み**：USBマスストレージモード経由でUF2ファイル

## ASPICEプロセス対応

本プロジェクトはASPICE V字モデルに準拠したプロセスで進める。

| プロセス | 内容 | 本プロジェクトでの対応 |
| ------- | --- | ------------------- |
| SYS.2 | システム要件分析 | `10_system_requirements.md` |
| SYS.3 | システムアーキテクチャ設計 | **省略**（ハードウェア構成は前提として本文書に記載） |
| SWE.1 | ソフトウェア要件分析 | `20_software_requirements.md` |
| SWE.2.1 | ソフトウェアアーキテクチャ設計 | `30_architecture.md` |
| SWE.2.2 | コンポーネント設計 | `40_component_design.md` （段階2.5-2 で作成予定） |
| SWE.3.1 | ソフトウェア詳細設計 | `50_detailed_design.md` （段階2.5-3 で作成予定、Doxygen 補完） |
| SWE.3.2 | 実装 | 段階3 |
| SWE.4 | ユニットテスト | 段階4-1 |
| SWE.5.2 | コンポーネントテスト | 段階4-2 |

| SWE.6 | 受け入れテスト | 段階4-4 |

### SYS.3 を省略する判断

本プロジェクトはハードウェアが既に決まっており（Raspberry Pi Pico）、ハードウェアとソフトウェアの分担を改めて設計する必要がない。
ハードウェア構成は上記「ハードウェア前提」セクションで明文化することで、SYS.3の文書化は省略する。

将来、ハードウェアの選択肢が複数になる場合（例：RP2350への移植検討）では、SYS.3として別文書を起こす可能性がある。

### テスト設計のタイミング

ASPICE V字モデルに従い、設計の各段階で対応するテストもペアで設計する：

- SWE.2.1（アーキテクチャ設計） → 統合テスト設計
- SWE.2.2（コンポーネント設計） → コンポーネントテスト設計
- SWE.3.1（詳細設計） → ユニットテスト設計

これらのテスト設計文書は段階2.5で作成し、段階4でテストを実施する。

## 段階構成

本プロジェクトは以下の段階で進める：

- 段階1：要件分析（SYS.2、SWE.1）
- 段階2：ベアメタル基盤構築（チュートリアル位置づけ）
- 段階2.5：設計フェーズ（SWE.2.1〜SWE.3.1）
- 段階3：実装フェーズ（SWE.3.2）
- 段階4：テスト実施フェーズ（SWE.4〜SWE.6）
- 段階5〜9：Rust移植
- 段階10：言語比較ドキュメント

進捗状況の詳細は [README.md](https://github.com/harukikiya/rp2040-temp-monitor/blob/master/README.md) を参照。
